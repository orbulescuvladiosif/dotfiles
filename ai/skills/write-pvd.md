# Write a PVD

A PVD captures the shape of a product — what it is, how people use it, what makes it distinct.

## What belongs

- What the product is and who it's for
- The core user interface at a conceptual level
- The core mechanic — the loop the user lives in
- Decisions that define the product's character

## Process

Start with what the product is. If it doesn't fit in a short paragraph, the vision isn't clear yet. Then describe the interface at the surface level. Then the core mechanic. Merge what's really the same idea. Cut requirements in disguise.

## Done when

Nothing feels out of place and nothing feels missing. Length follows content.
