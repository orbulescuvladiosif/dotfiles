# Git

- Follow Conventional Commits — atomic commits, scoped branches
- Keep commit subject ≤50 chars
- No commit body — subject only
- No Co-Authored-By or other commit trailers
- Scan recent commit history before committing — match the repo's style
- Use the repo PR template if present; else state what + why
- Keep PRs ≤600 production LOC (excl. tests, mocks, infra). Split only when each part works standalone
